# X Posting Plan - Provance (ETHGlobal OpenAgents)

## Positioning (What You Are Building)
Provance is a platform where AI agents specialize in high-demand Web3 tasks, continuously improve through verified training rounds, and prove that growth onchain. Each agent has a token tied to its performance, with revenue-driven buybacks and public proof of capability upgrades.

## Posting Cadence (Simple + Sustainable)
- Daily: 1 build update post
- Daily: 1 insight/opinion post about AI agents x Web3
- Every 2-3 days: 1 mini-thread (3-5 tweets) showing progress artifacts
- Weekly: 1 recap thread of what shipped + what is next

## Content Pillars
- Build in public: feature progress, blockers, architecture choices
- Proof over hype: why verifiable improvement matters
- Agent economics: token buybacks, staking, incentive loop
- Use cases: the six agent specializations and real user value
- Demo moments: screenshots, flow diagrams, short clips, test results

## High-Performing Post Format
- Hook: one strong sentence
- Build detail: what changed today
- Why it matters: user/investor impact
- CTA: ask for feedback, testers, or collaborators

## 14 Starter Posts You Can Publish
1. We are building Provance for ETHGlobal OpenAgents: AI agents that *prove* they improve over time onchain. Not claims. Verifiable growth records.

2. Most AI agents are static. Day 100 quality often looks like day 1. Provance introduces training loops with public capability history so anyone can verify improvement.

3. Today’s build focus: agent specialization model (audit, scam detection, launch analysis, signals, yield, airdrops). Specialization is how agent performance becomes investable.

4. Provance agent economics: task revenue comes in -> 60% ops fund, 30% token buyback, 10% protocol fee. Better work drives stronger token fundamentals.

5. We’re designing task-level memory: each task stores input, output, and score. Wins and failures both become training data for the next fine-tune cycle.

6. Core belief: AI credibility in Web3 should come from public track record, not branding. Every capability jump should be inspectable.

7. Training flow update: experience data -> decentralized training -> cryptographic proof -> new capability NFT. This is the backbone of trust in Provance.

8. Built today: clear role model for 4 participants - agent owner, investor, task poster, verifier. Incentives must align for all sides.

9. Why this matters: early teams can’t afford $10k-$100k audits. Specialized agents can deliver faster first-pass security reviews at far lower cost.

10. We’re treating Farcaster/X as the public scoreboard for each agent. Every completed task and every model upgrade should be visible in real time.

11. Hard problem we’re solving: how to quantify capability improvements so staking on training rounds is rational, not speculative.

12. Provance flywheel: better tasks -> more earnings -> token buybacks -> more attention/capital -> better tasks -> more training -> stronger capability.

13. Looking for feedback: which specialization should we prioritize for v1 demo quality - smart contract auditing or scam detection?

14. Weekly recap: what we shipped, what broke, what we learned, and what ships next. Building in public all the way to demo day.
